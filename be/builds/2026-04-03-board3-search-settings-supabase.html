Decisions Made

  • Region picker is multi-select: You can enable SoCal + Phoenix + Vegas simultaneously. CL locations merge and dedupe. More regions = wider net but higher Apify cost — noted in the UI.
  • Results-per-source splits across CL locations: If you set 15 results and have 3 CL locations, each gets ~5. Prevents runaway Apify calls.
  • Supabase schema uses room_code as FK: No UUIDs in the user-facing flow. 6-digit code is the only handshake.
  • Realtime enabled on item_overlays + activity only: Rooms and shared_comps don't need live push — they're queried on demand.
  • Client is safe without env vars: supabase.ts exports null if VITE_SUPABASE_URL is missing. App works without sync — just runs local-only like before.
