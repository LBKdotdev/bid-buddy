What Went Well

Mid-stream pivot handled cleanly. Started with Cloudflare D1, realized Joey wasn't available to own the backend, pivoted to Supabase in one conversation. Re-planned, re-estimated, zero wasted code. The D1 plan was superseded but the search settings work carried forward untouched.
Overlay-only sync was the right call. Only syncing status/maxBid/note/buddyTag keeps payloads tiny and avoids conflicts on heavy cached data (comps, estimates, issues). Each phone owns its own IndexedDB — Supabase is just the overlay layer.
Shared comps cache saves real money. If one buddy searches "2019 Can-Am Ryker 900", the result goes to Supabase. Other buddies skip the Apify call entirely. At ~$0.03/search across 500+ items and 3 users, that's meaningful savings.
Zero fix loops. ~B → ~T → ~S with no ~F cycle. TypeScript caught everything at compile time. The applyOverlay no-loop design (writes to IndexedDB directly, not via saveItem) avoided the classic real-time echo bug.
Board 5 Item 1 was already done. Offline queue shipped as part of Board 4's syncClient. Recognized it, marked it via external contributor rule, moved on. No duplicate work.

What Could Be Better

Supabase setup was manual and bumpy. Unpausing the project, finding API keys, running SQL, setting Vercel env vars — all required back-and-forth guidance with screenshots. A provisioner script or setup wizard would smooth this for any future Supabase project.
No live integration test. 12/12 checks are code-level verification (exports exist, wiring correct, types compile). Haven't tested actual room creation → join → overlay sync across two browser tabs yet. That's the real proof.
Bundle size growing. Production build is 950KB (gzip 269KB). Supabase JS SDK is chunky. May want to lazy-load the sync module or code-split Settings screen eventually.

What's Next

Live test with 2-3 phones. Open the app on two devices, create a room on one, join on the other. Change a status, confirm it appears. This is the acid test before auction day.
Floor screen sync awareness. FloorScreen is the live auction lookup — it should show a sync indicator per item (who marked it, when). Currently it reads from local IndexedDB but doesn't surface buddy edits visually.
Buddy tag assignment in room. When in a room, items could auto-assign a buddy tag showing who's covering which item on the floor. Prevents duplicate bidding.
Conflict resolution. Current model is last-write-wins. Fine for 3 buddies coordinating, but if two people edit the same item's maxBid simultaneously, one gets overwritten silently. A toast notification ("Joey changed maxBid on #142") would help — the activity feed exists but it's only in Settings.
