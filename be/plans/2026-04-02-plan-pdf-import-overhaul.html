PDF Import Overhaul

Plan — Awaiting Approval

Intent

Make Bid Buddy import every field of value from both NPA catalog PDFs — motorcycle and RVM — so Scotty has full data on the auction floor. Auto-detect which catalog is which from the filename. No manual catalog type picker.

Scope

What's broken

  • PDF upload always routes to motorcycle parser — no way to import RVM catalog
  • Motorcycle parser misses: zone, EM/FR/TR subscores, DMV penalties, book value (as structured field)
  • RVM parser misses: color, stock#, book value (reliable), detailed condition notes
  • InventoryItem type missing fields: color, zone, bookValue, conditionDetail
  • RVM items (Sea-Doo, Yamaha PWC, boats, trailers) never get category auto-detection because they never reach detectCategory()

Approach

Phase 1 — Extend the data model

Add new fields to InventoryItem: color, zone, bookValue (number), conditionDetail (string — summary of subscores). These are nullable. No migration needed — IndexedDB is schemaless.

Phase 2 — Auto-detect catalog type

In the PDF upload handler, detect from filename:

  • Filename contains "RVM" → rv_marine
  • Everything else → motorcycles

Replace the parsePDF(file) call with parseCatalogPDF(file, detectedType). Kill the legacy parsePDF wrapper.

Phase 3 — Rewrite motorcycle parser

The existing parser works for the basic fields but misses valuable data. Add extraction for:

  • zone — already captured in regex group, just not stored
  • bookValue — already extracted as number, store it properly
  • conditionDetail — capture the EM=9, FR=8, TR=9 string
  • DMV penalties — append to notes field

Run detectCategory() on parsed make/model so SxS items in the moto catalog get correct category.

Phase 4 — Rewrite RVM parser

Current RVM parser barely works. Rewrite to match the actual block format:

  • Title line: YYYY MAKE MODEL at top of each block
  • Structured fields: LOCATION, DOCS, CONDITION REPORT, VIN, STOCK#, Mi/HR, COLOR, BOOK VALUE, AUCTION #
  • conditionDetail — summarize the low-scoring items (score < 7) as a quick-read string
  • Run detectCategory() on make/model — Sea-Doo → rv_marine, trailers → rv_marine, etc.

Phase 5 — Test with real PDFs

Import both real NPA PDFs. Verify item counts, field completeness, category detection. Check edge cases: multi-line models, TMU mileage, missing book values, CPO candidates (#).

Phase 6 — Clean up dead Supabase

Remove supabase/ folder, @supabase/supabase-js from package.json, stale references in CLAUDE.md.

Constraints

  • No UI changes to import flow — just pick a file, it works
  • Existing CSV import must keep working
  • Existing items in IndexedDB must not break — new fields are nullable
  • No new dependencies
  • Must work offline (pdfjs-dist already bundled)
