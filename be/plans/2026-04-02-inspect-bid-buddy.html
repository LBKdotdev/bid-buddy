Demo Blockers for Tomorrow

1. Category Hardcoded to Motorcycles

All CSV/API imports stamp every item as motorcycles regardless of make/model. At a mixed auction (ATVs, SxS, RV/Marine, Golf), this means wrong labels and potentially wrong comps context. The defaultCategory setting exists in the UI but is never consulted during import.

Fix: Auto-detect category from make/model at import time. Can-Am Ryker = motorcycle. Polaris RZR = ATV/SxS. Sea-Doo = RV/Marine. EZ-GO = Golf. Default fallback for unknowns.

2. Dead Code — Unused Screens

4 screens not wired into App.tsx navigation: ItemDetailsScreen (duplicate), InventoryScreen, AuctionDataScreen, KittyCompsScreen. Plus ItemRow.tsx component unused. Not a blocker but adds confusion.

3. Comps Source Toggles Duplicated

Settings screen has global source toggles (eBay/CycleTrader/Craigslist). ItemDetailScreen has local overrides stored in localStorage. User can't tell which takes precedence.

4. No Error Boundaries

A crash in any screen (OCR failure, comps timeout) takes down the entire app. One React error boundary around the tab content would prevent this.
