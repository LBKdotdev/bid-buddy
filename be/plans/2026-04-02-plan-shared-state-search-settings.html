Plan: Shared State + Search Settings

Bid Buddy — Multi-user sync + configurable comp search regions

Plan — Awaiting Approval

Intent

Three people on the auction floor, same inventory, same day. Right now each phone is an island — edits, notes, max bids, and comps don't travel between devices. When Scotty marks a Teryx "interested" with a $4,200 max bid, Joey and the third person don't see it. That's a problem at speed.

Secondary: comp searches are locked to hardcoded regions and result counts. The team needs to dial in radius, pick regions, and control how many results come back — especially when Apify dollars are on the line.

Scope

In Scope

  • Shared inventory state — status, notes, maxBid, buddyTag sync across devices in real-time
  • Shared comps cache — one person runs a deep search, everyone benefits
  • Search settings — radius slider, region picker, results-per-source control
  • Conflict handling — last-write-wins with user attribution ("Joey set max bid $4,200 at 2:14pm")
  • Offline resilience — keep working if signal drops, sync when back

Out of Scope

  • User accounts / auth — this is a team tool, not a public app. Shared key or room code is enough.
  • Chat / messaging between users
  • Inventory write conflicts on import (import is always additive/merge, same as now)
  • Moving off Vercel — keep the existing API functions where they are

Approach

Backend: Cloudflare D1 + Workers

Joey's stack preference. D1 is SQLite at the edge — fast reads, global, free tier covers this use case. One Worker handles the sync API. Tables:

TablePurposeKey Fields
roomsTeam sessionroom_code, created_at, expires_at
itemsShared inventory stateid, item_number, status, max_bid, note, buddy_tag, updated_by, updated_at
comps_cacheShared comp resultsquery_hash, results_json, fetched_at, fetched_by
activityChange logitem_id, field, old_value, new_value, user, timestamp

The items table stores only the overlay — status, notes, maxBid, buddyTag. The full inventory (year, make, model, VIN, etc.) stays in IndexedDB. This keeps the sync payload tiny and fast.

Sync Model: Poll + Push

  • On edit: POST change to D1 immediately → update local IndexedDB
  • Background poll: Every 10 seconds, GET changes since last sync timestamp
  • On app open: Full pull of room state → merge into local DB
  • Offline: Queue edits in localStorage → flush when back online
  • Conflict: Last-write-wins by updated_at. Activity log preserves history.

Room System (No Auth)

On Settings screen: "Create Room" generates a 6-digit code. Others enter it to join. Room expires after 24 hours (covers a full auction day). No passwords, no logins — just a code you share verbally on the floor.

Search Settings

Add to the existing Settings screen under Comps Lookup:

  • Search Radius: Slider 50–300 miles (currently hardcoded 150). Sent as radius param to fetch-comps.
  • Regions: Multi-select checkboxes — SoCal, NorCal, Phoenix, Vegas, Dallas, Houston, Orlando. Currently auto-detected from zip. Let user override.
  • Results per source: Slider 5–50 (eBay max, CL limit per location, FB resultsLimit). Currently hardcoded 25/10/15.
  • Persist in settings — flows through to /api/fetch-comps as new POST body fields.

Constraints

  • D1 free tier: 5M reads/day, 100K writes/day, 5GB storage. More than enough for 3 users, 1000 items, 1 auction day.
  • No auth: Room codes are security-by-obscurity. Fine for a team tool — not for public release.
  • Sync is overlay-only: Full item data (VIN, year, make, model, photos) stays local. Only status/notes/maxBid/buddyTag sync. Keeps payloads small, avoids import conflicts.
  • Last-write-wins: If Joey and Scotty edit the same item's note at the same time, the later write wins. Activity log preserves both versions for recovery.
  • 10-second poll: Not true real-time (would need WebSockets). Acceptable for auction floor pace — you're not trading stocks.
  • 24-hour rooms: Auto-expire. No long-term state management. Each auction day is a fresh room.
  • Existing Vercel API stays: fetch-comps, fetch-npa-inventory, scan-tag remain on Vercel. D1 Worker handles only sync. Two hosts, clean separation.
